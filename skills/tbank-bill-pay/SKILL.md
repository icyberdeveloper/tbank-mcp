# Оплата счетов через T-Bank MCP

## Инструменты

| Тул | Что делает |
|---|---|
| `get_data("providers")` | Список доступных провайдеров |
| `get_data(section)` | Разные данные (requisites — реквизиты счёта) |
| `payment_commission(body)` | Предпросмотр комиссии |
| `transfer(amount, to, desc)` | Перевод (для пополнения телефона — P2P) |
| `list_accounts()` | Счета + балансы (выбрать откуда платить) |

## Типы платежей и как их проводить

### Пополнение телефона

**«Пополни МТС на 500» / «закинь телефон»**

1. Если получатель — номер телефона → это P2P перевод:
   ```
   transfer(500, "+79991234567", "Пополнение МТС")
   ```
   Бесплатно (P2P внутри Т-Банка).

2. Если нужен провайдер (не Т-Банк номер):
   - Провайдер: `mts`, `beeline`, `megafon`, `tele2`, `yota`
   - Формат: `transfer(500, "<phone>", "МТС")`

### Интернет

**«Оплати интернет Ростелеком»**

1. Узнай у пользователя:
   - Номер договора / лицевой счёт
   - Сумму

2. Провайдер: `rtk` (Ростелеком), `domru` (Дом.ru), `akado` (АКАДО), `mts-internet`

3. Проверь комиссию:
   ```
   payment_commission({
     "payParameters": {
       "provider": "rtk",
       "moneyAmount": 1000,
       "currency": "RUB",
       "paymentType": "Payment",
       "providerFields": {
         "account": "<номер договора>",
         "pointerType": "ACCOUNT"
       }
     }
   })
   ```

4. Покажи: «Ростелеком, договор 123456, 1000₽, комиссия 0₽. Оплатить?»
5. После подтверждения → `pay(body)` или `transfer`.

### ЖКХ (коммуналка)

**«Оплати коммуналку» / «оплати ЖКХ»**

1. Уточни что именно:
   - «Что оплатить: газ, электричество, вода, отопление, или всё вместе?»
   - Если пользователь не знает → предложи поискать по региону/адресу.

2. Узнай: лицевой счёт / номер плательщика + сумму.

3. Провайдеры ЖКХ по типу:
   - Газ: `mosoblgaz`, `mosgaz`
   - Электричество: `mossvet`, `energy`
   - Вода: `mosvodokanal`
   - ЕПД (единый платёжный документ): `epd-moscow`

4. Комиссия → подтверждение → оплата.

### Налоги (ФНС)

**«Оплати налоги» / «заплати налог на имущество»**

1. Узнай: ИНН или индекс документа (УИН) из квитанции.
2. Провайдер: `nalog`, `fns`
3. Если есть УИН — проще: оплата по УИН без ИНН.
4. Комиссия → подтверждение → оплата.

### Штрафы ГИБДД

**«Оплати штраф» / «проверь штрафы»**

1. Узнай: номер ВУ или госномер авто, или номер постановления (УИН).
2. Провайдер: `gibdd`, `gibdd-online-rf`
3. Проверь наличие штрафов (если номер ВУ/госномер известен).
4. Комиссия → подтверждение → оплата.

### Образование

**«Оплати садик» / «оплати школу»**

1. Узнай: лицевой счёт ребёнка / номер договора.
2. Провайдер: зависит от региона (`edu-moscow`, `detsad`, etc.)
3. Комиссия → подтверждение → оплата.

## Флоу (общий)

```
1. Определить тип платежа (телефон/интернет/ЖКХ/налог/штраф/образование)
2. Узнать у пользователя:
   - Номер счёта / договора / телефона / ВУ / УИН
   - Сумму
3. Найти провайдера (get_data("providers") + фильтр по имени)
4. Проверить комиссию (payment_commission)
5. Показать пользователю: провайдер + счёт + сумма + комиссия
6. Подтверждение пользователя — ОБЯЗАТЕЛЬНО
7. Оплата (transfer для телефона, pay для услуг)
8. Отчёт
```

## Правила

1. **Всегда проверяй комиссию** перед оплатой.
2. **Всегда подтверждай** с пользователем: провайдер + номер счёта + сумма + комиссия.
3. **Не угадывай сумму** — спроси. Если пользователь не знает → предложи проверить.
4. **Лицевой счёт / договор** — обязательный реквизит для ЖКХ/интернета/образования. Без него платить нельзя.
5. **УИН** — для налогов/штрафов. Удобнее чем ИНН.
6. **Пополнение телефона** — чаще всего бесплатный P2P внутри Т-Банка.
7. **Сохраняй провайдера** — если пользователь платит ЖКХ каждый месяц, запомни провайдера + счёт для следующего раза.
